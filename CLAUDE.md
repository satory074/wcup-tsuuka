# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 概要

ワールドカップ グループステージの **通過タイムライン** 可視化サイト（Astro 5 + TypeScript + Tailwind v4 + GitHub Pages 静的サイト、`base: /wcup-tsuuka`）。主役は**タイムライン＝順位バンプチャート（SVG 折れ線）**（縦軸＝順位1〜4・横軸＝イベント時系列・チームごとに1本の色付き折れ線で順位推移を描く。全節が1画面に収まり横スクロール不要、右端に最終順位の国旗+国名、点＝得点で動いた瞬間でツールチップに得点者）。**タイムラインは単一**（全試合の分刻みゴール）＝表示モード切替は廃止。各節（matchday）が完全消化されると、その節の終了時点に**節末スナップ（◇＝中空リングの頂点）**を1つ差し込み、**対戦した両チームそれぞれのレーン上にその節の試合スコアをチャート上に直接描く**（`.tl-round-score`・1スコアは両参加チームのレーンに2回出る＝各レーンに「自分が戦った試合結果」が乗り、対戦していない第3チームのレーンと重ならない・トリコードは各チームの線色で色分け）。得点点は前半/後半を問わず一律で強調丸（`is-scorer`・●）。チャート下には**得点タイムライン（試合別カラム・常設 `<section.tl-log>`＝1節2カラム＝試合ごとに分割・節グループは縦積み＝各節が次の段）**＝節グループ `.tlog-md-group`（節見出し＋ `.tlog-md-cols` の2試合）の各 `.tlog-col` が 対戦見出し `.tlog-match`（国名＋最終スコア）→各ゴール（時刻・⚽得点者・国名＋途中スコア）。**国名表記は全表示に適用**（日程・結果カード・決勝トーナメント・順位表・ランキング・チャート右端ラベル・ツールチップ。例外＝チャート上の節結果スコアチップ `.tl-round-score` のみ省スペースで略号のまま）。FIFA世界ランキングは**順位表に併記**（`(FIFA ◯位)`）し、さらに**右レールにその大会時点の世界ランキング一覧パネル（`#fifa-ranking`＝全約211カ国を順位順・出場国を強調／非出場国は淡色）**を出す。**ページは2カラム共通シェル（`.layout-grid`・一覧/詳細で同一）**＝**左カラム `.layout-main`（主筋＝scope内容＋決勝トーナメント `#knockout`）／右レール `.layout-side`（得点ランキング→FIFAランキングを縦積みした `#rankings`）**。detail の左カラムは **日程・結果（`#schedule`・最上部）→ 最終順位（`#detail-main`）→ 通過ステータス（`#status`）→ タイムライン（`#detail-timeline`＝チャート＋試合別得点ログ）→ 決勝トーナメント** の縦並び。**右レールは補足情報の定位置（UI ベストプラクティス）／決勝トーナメントは主筋なので左**。**3位比較は一覧のみ・通過条件（シナリオ）パネルは廃止**（`#schedule` は一覧・詳細で共通＝早期 return 前に1回描画）。**日程・結果（`#schedule`＝全試合＋決勝トーナメントの横並びカルーセル `.sched-carousel`・一覧/詳細とも左カラム最上部）**は全グループの全試合（2022:48／2026:72）＋決勝トーナメント（2018/2022:16／2026:32）を**キックオフ時系列**で1枚=1試合のカード（日時＋組バッジ／KOはラウンドバッジ `.sched-ko-badge`＋home/away の旗・国名・得点／未消化は `–`）に並べ横スクロール、**該当グループを強調**（`.is-current`）し初期表示でそこへ自動横スクロール。**グループカードは `<button data-action="drill-group">`＝クリックでそのグループの詳細へドリル**（KOカード `.sched-card.is-ko` はグループ非依存＝非クリック・未確定の対戦はスロットラベル "M73 勝者"/"3位…"）。KOカードの kickoff/組み合わせは `knockoutSchedule`（後述）由来。

**決勝トーナメント ブラケット（`#knockout`）は scope 非依存＝一覧・詳細の両方で全幅常時表示**（`#overview`/`#detail-view` の兄弟）。詳細は `## 決勝トーナメント ブラケット` を参照。

**表示範囲（scope）= 2つ**（ヘッダ下「一覧／詳細」トグル。**既定は overview**＝最新大会 2026 の一覧。URL は既定 overview を省略し `?scope=detail` を明示）:
- **detail** = 左カラム `.layout-main` に **日程・結果（`#schedule`・最上部）→ 最終順位（`#detail-main`＝順位表〔FIFA順位併記・`max-width:660`〕＋通過ステータス）→ タイムライン（`#detail-timeline`）→ 決勝トーナメント（`#knockout`）** の縦積み。得点＋FIFAランキングは右レール `.layout-side` の共通 `#rankings`（後述）に分離。**3位比較・通過条件パネルは detail に出さない**（3位は一覧のみ・通過条件は廃止）。`?group=` の共有URLを温存。
- **overview**（一覧・既定）= 全グループのコンパクト順位表カードをレスポンシブグリッド表示（NBC/ESPN 型）。各カード＝組文字＋**進行フェーズバッジ**（確定/最終節/進行中）＋4行ミニ表（順位・国旗・略号・得失点差・勝点。上位2緑＋カットオフ点線・🎲抽選・未消化は「– –」）。2026 はグリッド下に全幅でベスト3位表（既存 `bestThirdsHTML` 再利用）。カードクリック（`data-action="drill-group"`）でそのグループの **detail** へドリルイン。一覧は `Standings` の純投影＋ランキング（scorers）のみで、`groupStatus`/timeline は detail のときだけ計算する（フェーズバッジは消化試合数からの安価な `derivePhase` で導出）。日程・結果は一覧でも左カラム最上部に表示（クリックでドリル）。
- **2カラム共通シェル** = `.layout-grid`（`grid-template-columns: minmax(0,1fr) 340px`・一覧/詳細で常設・狭幅980pxで縦積み）。`.layout-main`（左）に `#overview`/`#detail-view`/`#knockout`、`.layout-side`（右レール）に `#rankings`。`#knockout`・`#rankings` は scope 非依存で `rankingsHTML(view)`/`knockoutHTML(view)` を早期 return より前に1回描画＝両ビューで同一。
- **モバイル（≤980px）= 結論優先に並べ替え（CSS のみ・DOM 不変）**: `@media(max-width:980px)` で `.layout-grid` を `display:flex;flex-direction:column`、`.layout-main`/`.layout-side` を `display:contents` に畳み、子セクションに `order` を付与＝視覚順を **詳細内容（最終順位→チャート→得点ログ）→ ランキング → 決勝T → 日程** に再構成（DOM 順は不変なので domtest の順序アサーション・共有URL・SEO に影響しない）。**≤640px**: 主役チャートを幅フィット（`.tl-chart-wrap{min-width:0}`・右端の最終順位＝結論が横スクロール無しで見える）／節末スコアチップ `.tl-round-score` は得点ログと重複するため非表示／`.tl-dot` の `r` を拡大しタップ域確保／決勝T `.ko-bracket{flex-direction:column}` で縦積みリスト化／FIFA 211カ国表は `#fifa-ranking .fr-card{max-height:65vh;overflow:auto}` で内部スクロールに収める。スケジュール自動横スクロール（`render.ts`）は `matchMedia('(min-width:981px)')` のときだけ＝モバイルでは読み位置を奪わない。
- **アクセシビリティ**: 先頭にスキップリンク（`.skip-link`→`#main-content`＝`.layout-main`）。チャートの**得点者・試合結果はホバー専用（SVG `<title>`）だけでなく、得点点 `.tl-dot.is-scorer`／節末点 `.is-roundend` を `tabindex=0`+`aria-label` で focus 可能にし、tap/focus で読み取り行 `.tl-readout`（`aria-live`）へ表示**＝タッチ/キーボードでも取得可能。連動ハイライトは `pointerover` に加え `focusin` でも発火（凡例 `.tl-leg-item` も `tabindex=0`）。折れ線は色だけでなく **`stroke-dasharray`（実線/破線/点線/一点鎖線）で冗長化**（色覚配慮・凡例の見本も同じ線種を SVG line で反映）。表ヘッダに `<th scope="col">`。淡色トークン `--ink-faint` は WCAG AA(4.5:1) を満たす濃さ（#586478）。ホバー減光は 0.3（過減光を回避）。
- **`#rankings`（右レール）** = 得点ランキング→FIFAランキングを**縦積み**（`.rankings-stack`>`.rankings-panel` id=`#top-scorers`/`#fifa-ranking`）。ホバー連動（`.is-hl`）は `#detail-view` と `#rankings` の両領域で `data-team` 同期。

**3大会を切替表示**（ヘッダの大会タブ＝`?cup=2018|2022|2026`、既定は **2026**＝最新大会）:
- **2018方式**（32カ国・8組 A–H・各組上位2通過）= 2018年ロシア大会の全48試合・実スコア＋全48試合のゴール分刻み（得点選手名つき）。**キックオフは全試合モスクワ時間(MSK)に正規化**（露は5時間帯にまたがるため、単一タイムラインの絶対時刻順を保つ）。**組Hはフェアプレー通過の実データケース**: 日本とセネガルが勝点4・総得失点差0・総得点4・直接対決2-2で完全同値→警告数（日本4枚 < セネガル6枚）で日本が2位通過。これを表現するため組Hの全6試合に `cards` を投入し、`standings.ts` が**基準g フェアプレーを抽選の前に適用**する（順位決定ロジック節参照）。形式は2022と同一なので新データファイル1つ＋薄い配線（`Cup`型／`DATA`／`CUPS`）で追加。
- **2022方式**（32カ国・8組 A–H・各組上位2通過）= 2022年カタール大会の全48試合・実スコア＋全48試合のゴール分刻み（得点選手名つき）。
- **2026方式**（48カ国・12組 A–L・各組上位2 ＋ **全12組の3位の上位8** が R32 へ）= 2026年北中米大会。**大会進行中につき消化済み試合のみ実スコア**（残りは未消化）。`engine/thirds.ts` の `computeBestThirds` が3位の横断ランキングを算出し、順位表の下に**「3位チーム比較」パネル**（上位8緑・進行中は暫定/抽選）を出す。

エンジンは形式非依存（組数 8/12 は構造から導出）。大会切替は `?cup=` を書いて全再読込（リスナー重複回避・state リセット）。

```bash
npm install
npm run dev        # http://localhost:4321/wcup-tsuuka/
npm run build      # dist/（型チェック込み）
npm run typecheck  # astro check
npm run test       # tsx scripts/smoketest.ts && tsx scripts/domtest.ts
```

## アーキテクチャの鉄則（厳守）

- **`src/engine/` は純TS**: DOM・Date・フレームワークに触れない。型は `engine/types.ts` に集約。
- **DOM に触れるのは `src/app/`（`render.ts` / `main.ts`）と `src/pages/` のみ**。`render.ts` が唯一の描画層で、イベントはルートの click を `data-action` 委譲で捌く（kisei/moshirasu パターン）。ホバー連動ハイライトも `pointerover`/`pointerleave` 委譲＝`data-team` を持つ要素（チャートの線/点/右端ラベル・凡例・順位表行）をまたいで `.is-hl`/`.is-hovering` を同期。
- **データ検証は手書き `engine/validate.ts`（Zod 不使用）**。順位・タイムラインをクライアントで計算するためエンジンをバンドルする → Zod は載せない。検証は smoketest と boot の両方で走る。**形式非依存**: 組数 G は 8 か 12、`teams==4G`・`matches==6G`・各組4チーム6対戦を構造から検証（32/8/48 をハードコードしない）。ラウンドロビン検査は宣言済みの組だけを回す（`GROUP_IDS`＝A–L の宇宙は直接ループしない）。
- `GroupId`/`GROUP_IDS` は A–L（12組）。`compile.ts` は宣言された組（`t.groups`）から `ct.groups` を導出するので 2022→A–H / 2026→A–L に自動追従（タブ・既定組もこれに従う）。
- 内部リンクは `src/lib/url.ts` の `siteLink()`（`import.meta.env.BASE_URL`）。**`<head>` の手書きアセットリンク（favicon・OG画像）も必ず `siteLink()` 経由**＝`${import.meta.env.BASE_URL}foo` の直結は `trailingSlash:"ignore"` で BASE_URL に末尾スラッシュが無く `/wcup-tsuukafoo`（404）になる（実バグだった。Astro 生成のCSS/JSリンクは自前で正しく付くが手書きは付かない）。`src/app/url.ts` が表示状態⇔クエリ（`cup`/`group`/`scope`。**既定大会は 2026・既定 scope は overview**。既定 `scope=overview` は URL に出さず `scope=detail` を明示する。旧 `view`/`pivot`/`assume` は廃止＝付いていても decode が無視。**注意**: 旧 detail 共有URL（scope無し＝従来 detail）は新既定 overview として開く＝group は保持。旧 `?scope=overview` は既定なので無視＝overview のまま）。
- **OG/サムネ画像**: `public/og.png`（1200×630・順位バンプチャート意匠、ソースは `public/og.svg`）を `<head>` の `og:image`/`twitter:image`（絶対URL＝`new URL(siteLink("og.png"), import.meta.env.SITE)`）で配信。これが基盤サイト basecamp の「作品」一覧サムネにも使われる（basecamp 側が homepage の og:image を自動抽出。`og:image` が無いと汎用 placeholder になる）。`og.svg` を変えたら basecamp の sharp 等で `og.png` を再ラスタライズ。
- `astro.config.mjs` の Tailwind v4 プラグインは型不一致回避で `any` キャスト。配色は全て CSS 変数トークン（`--ts-*`=タイプスケール／`--sp-*`=スペーシング／色トークン）に集約。既定はライトで `color-scheme: light dark`、OS ダーク時は `@media (prefers-color-scheme: dark)` が色トークンだけを暗色に差し替える（ブラウザ強制反転の濁りを避けるため自前ダークを持つ）。

## データ

大会ごとに JSON が単一の真実。`src/app/main.ts` が3つを import し `?cup` で選ぶ（`DATA: Record<Cup, unknown>`、既定 2026）。
- `src/data/worldcup2018.json`（`meta` / `teams[32]` / `groups[8]` / `matches[48]` / **`knockout[16]`** / **`knockoutSchedule[16]`**、`advancePerGroup:2`）。**全48試合に goals 投入済み**（消化済み大会、0-0 は `[]`）。**kickoff は全試合 MSK 正規化**。**組Hの6試合のみ `cards`**（日本＝黄4／セネガル＝黄6＝フェアプレー通過の根拠）。**`knockout` は決勝T全16試合の score/winner/goals**（優勝フランス・得点王ケイン6）。**`knockoutSchedule` は決勝T全16試合の kickoff**（テンプレ id "r16-1".."f" キー）。出典は Wikipedia 各組/knockout ページ（ja）。
- `src/data/worldcup2022.json`（`meta` / `teams[32]` / `groups[8]` / `matches[48]` / **`knockout[16]`** / **`knockoutSchedule[16]`**、`advancePerGroup:2`）。**全48試合に goals 投入済み**（0-0 は `[]`）。**`knockout` は決勝T全16試合の score/winner/goals**（優勝アルゼンチン・得点王ムバッペ8）。**`knockoutSchedule` は決勝T全16試合の kickoff**。
- `src/data/worldcup2026.json`（`teams[48]` / `groups[12]` / `matches[72]` / **`knockoutSchedule[32]`**、`advancePerGroup:2` ＋ **`advanceBestThirds:8`**）。組分け・日程は2025/12の本抽選ベース。**消化済み試合のみ score**（残りは score 省略＝未消化）、goals（得点者・分）は全72試合に投入済み（2026-06-28 時点で全12組が第3節まで消化＝グループステージ完了・全組決着）＝全組でライブ年表が有効。**`knockoutSchedule[32]`（id "73".."104"）= R32〜決勝の kickoff ＋ R32 の3位8枠の割当（`third`）**。KOの結果（`knockout`）は**R32 開幕（2026-06-28〜）につき消化済み試合のみ追記中**＝2026-06-29 時点で **M73（南アフリカ 0-1 カナダ・ユースタキオ 90+2'）のみ確定**（試合消化のたび `knockout` に1試合ずつ追記する運用）。出典は Wikipedia 各組／"2026 FIFA World Cup knockout stage"（直近試合は ESPN 等でクロス検証）。
- `src/data/flag-colors.json`（タイムライン線色用）＝ team id → **国旗の主要色 HEX の順序付き配列**（識別性の高い順・白/黒は後ろ）。全3大会の全 team id（58カ国＝2018追加で rus/per/isl/nga が加わった）を網羅。`engine/validate.ts` の `validateFlagColors(raw, teamIds)` が HEX 形式＋**カバレッジ**（全 id に色がある）を強制＝2026 に新チーム追加時に色未登録なら smoketest→CI で落ちる。出典は flagcolorcodes.com / Wikipedia 各国旗。線色の算出は `src/app/flagColors.ts`（タイムライン節参照）。
- `src/data/fifa-rankings.json`（FIFAランキングパネル用）＝ `{ "2018":[...], "2022":[...], "2026":[...] }`、各大会**全約211カ国**の `{rank, code, name(日本語), flag}`（その大会直前リリースの世界ランキング全表）。code は FIFA トリコード小文字（出場国は `teams[].id` と一致）。`render.ts` が `import` して `#fifa-ranking` パネルを描画（出場国は team データで強調・他は淡色）。`engine/validate.ts` の `validateFifaRankings(raw, participantsByCup)` が 構造＋rank/code 一意＋**出場国カバレッジ**を強制。smoketest が追加で**出場国の rank == teams[].fifaRank** をクロスチェック（データ二重管理の整合保証）。出典は FIFA公式／Wikipedia "FIFA Men's World Ranking"／fifaranking.net。

team id は小文字 FIFA トリコード（表示用は大文字化）。`matches[].score` 省略/null = 未消化。`cards`（`{home:{y,r}, away:{y,r}}`）は任意で、あると `standings.ts` が**基準g フェアプレー**を抽選の前に適用する（2018 組Hのみ投入・2022/2026 は皆無＝従来どおり）。`matches[].kickoff`（必須）= ISO 現地時間 `"2022-11-23T16:00"`（2018 は MSK 正規化）。タイムラインの絶対時刻並べ替え＆日時帯表示用（validate が形式強制）。`matches[].goals`（任意）= `{minute, plus?, side, player?}` の配列で**分刻みタイムライン**用。あれば**本数==score を validate が強制**（転記ミス検出）。`player` は得点選手（日本=漢字・他=カタカナ・OG は "名前(OG)"＝side は得点が記録された側、PK は "名前(PK)"）。`teams[].fifaRank`（任意・正整数）= 大会ごとの時点の **FIFA世界ランキング**（2018=2018年6月／2022=2022年10月／2026=2026年6月の大会直前順位）。順位表・一覧カードに併記する（validate が正整数を強制）。`meta.advanceBestThirds`（任意・省略時0）= 2026の「3位上位N通過」を駆動。**`knockout`（任意・トップレベル配列）= 決勝Tの実結果**＝`{round, home, away, score, winner:"home"|"away", shootout?, goals?}`（home/away は teamId、score は90分+延長＝PK戦は含めず winner/shootout で決着、goals は score と本数一致を validate が強制）。2018/2022 は全16試合投入・2026 は R32 進行中につき消化済み分のみ（2026-06-29 時点で M73 の1試合）。**`knockoutSchedule`（任意・トップレベル配列）= 決勝Tの日程**＝`{id, kickoff, third?}`（id は `bracketTemplate` の `KoMatch.id`＝2026 "73".."104" / 2018・2022 "r16-1".."f"、kickoff は ISO 現地、`third` は2026 R32 の「3位枠」に入る確定済みチーム teamId＝8件のみ）。`computeKnockout` が id で kickoff を `KoResolvedMatch.kickoff` に付与し、`third` で3位スロットを実チームに解決（**FIFA 495通り表は実装せず確定済みの1通りだけをデータ化**）。validate が id 重複・kickoff 形式・third のチーム存在を強制。`compileTournament()` が `validateTournament()` を通して `CompiledTournament`（Map 索引・`knockout`/`knockoutSchedule` は無ければ空）にする。

## ベスト3位（`engine/thirds.ts`）= 2026方式

`computeBestThirds(ct, standingsByGroup)` が各組3位（`rankGroups` で rank=3 クラスタ抽出）を横断ランキング。FIFA 順 = 勝点 → 総得失点差 → 総得点 → フェアプレー（`fairPlayPoints` を3試合合算、無ければタイ） → **抽選（teamId 昇順で決定的）**。head-to-head は不適用（3位同士は未対戦）。上位 `advanceBestThirds` 圏に**ブロック全体が収まるときだけ** `advances`、枠線を跨ぐ抽選は両者 false。**捏造しない**: 組内3位がタイ or 組が未完なら `state:"contention"`＋`undecided` で表面化（描画は「暫定/抽選」バッジ）。per-group の `advances`（上位2のみ）には混ぜず横断オーバーレイとして別管理（status・2022 の不変性を保つ）。**3位比較は一覧（overview）のみ**＝`overviewHTML` の `.overview-bt`（`advanceBestThirds>0` のときだけ中身を出す＝2022 は空）。detail には出さない。

## 決勝トーナメント ブラケット（`engine/knockout.ts`）

ノックアウトの**組み合わせ**を表示。ブラケット**テンプレートはエンジン定数**（フォーマット由来）で、`bracketTemplate(ct)` が組数で分岐＝12組(2026)→`R32_2026`（R32 M73-104。出典 Wikipedia "2026 FIFA World Cup knockout stage"）／8組(2018/2022)→`R16_8GROUP`（標準32カ国ブラケットのツリー）。**2018/2022 は JSON トップレベル `knockout` 配列でKO実結果を持つ**（各試合 `{round, home, away, score, winner, shootout?, goals?}`。出典 Wikipedia 各大会 knockout stage。2026 は R32 進行中＝消化済み分のみ追記中＝現状 M73 のみ）。**日程は別系統 `knockoutSchedule`**（id キーの kickoff＋2026の3位割当 `third`。前述データ節）。`computeKnockout(ct, standingsByGroup)` が各スロットを解決：`winner`/`runnerup` は対象組が**全消化済み(`played===3`)かつ当該 rank が単独確定**のときのみ実 `teamId`、それ以外は `undecided` でスロットラベル（"1A"/"2B"）。`third`（「勝者vs3位」8枠）は **`knockoutSchedule.third` があれば実 `teamId` に解決**（2026は確定済み8枠を割当＝R32 全32枠が実チーム）、無ければ集合ラベル（"3位 A/B/C/D/F"）のまま＝**FIFA 495通り表は実装しない**。`winnerOf`/`loserOf`（R16以降）は **KO結果があれば勝者/敗者の実 `teamId` に解決**（テンプレを依存順1パスで処理し `winnerById`/`loserById` を蓄積、スロット解決後のチームペアで `ct.knockout` を突き合わせ）。KO結果が一部のみの大会（2026）は、消化済み試合の勝者だけが次ラウンドのスロットを実チームに解決し（例: M73 勝者カナダ→R16 M90 の片側）、未消化分は未確定（"M75 勝者"）のまま。各 `KoResolvedMatch` に `kickoff`（schedule由来）・`result`（side1/side2 スコア・勝者側 `winnerSide`・PK戦）を持つ。**不変条件**: `teamId` があれば `undecided=false`／無ければ `true`（捏造しない）。`render.ts` の `knockoutHTML()` が `#knockout`（`#overview`/`#detail-view`/`#rankings` の兄弟・**一覧/詳細の両方で常時表示**・早期returnより前に更新）にラウンド列ブラケットを描画（各試合カードの見出し行 `.ko-head` に kickoff の **日付(曜日)＋時刻** を併記＝左に試合番号 `.ko-no`／右に `.ko-when`＝`.ko-date`「M/D(曜)」＋`.ko-time`「HH:MM」。曜日は `render.ts` の `fmtKickoff` が **Sakamoto のアルゴリズム（Date 不使用）** で算出）。**ブラケットはメインコンテンツ幅に収める**＝`.ko-bracket{width:100%}`＋各ラウンド列 `.ko-round{flex:1 1 0;min-width:96px}` でコンテナ幅を均等分割（横スクロールしない／狭幅では min-width 割れで `.ko-scroll` がスクロール）。確定枠は旗+国名（`data-team`＝`#knockout` 専用 hover で `.is-hl` 連動）、**消化済みは勝者を緑強調（`.ko-side.is-winner`）＋スコア併記（`.ko-score`）＋PK戦「PK 4-2」（`.ko-so`）・敗者は淡色**、未確定は破線スロット。**3位プール（`.ko-pool`）は「3位枠が未割当のときだけ」併記**（割当済み＝2026現在は非表示）。**将来**: 495表での3位厳密割当は out of scope（確定後は `knockoutSchedule.third` でデータ化）。

## 順位決定ロジック（`engine/standings.ts`）= 正しさの核

FIFA 2022 順:
1. 総勝点 → 2. **総**得失点差 → 3. **総**得点
→ ここで並んだチーム**同士の対戦のみ**で: 4. 勝点 → 5. 得失点差 → 6. 得点
→ 7. フェアプレー（カード少。`cards` データがある面子のみ） → 8. 抽選

**落とし穴**:
- step2-3 は**総合**（h2h より先）。よくあるバグ＝h2h GD を総得点より先に適用。→ smoketest が **組H（韓国 vs ウルグアイ：勝点・GD 同で総得点 4 vs 2）** で担保。組E は esp vs ger を総 GD で担保。
- h2h は「現在並んでいる面子だけ」の対戦で計算（`headToHead(cluster, ...)` は cluster を引数で受けキャッシュしない）。並びが変わると面子も変わるため、タイムラインの各スナップは順位を**ゼロから再計算**する（`viewAt` がその時点のスコアで `computeStandings` を呼ぶ）。
- **step7 フェアプレー（基準g）**は h2h（6）でも分割できなかったクラスタに、抽選の前に適用する（`resolveCluster` 内 `splitByFairPlay`）。`teamFairPlay` が**消化済み試合すべてに `cards` があるチームのみ**確定値を返し（1試合でも欠落＝undefined＝そのチームは基準g 不適用→抽選へ）、イエロー=1・レッド=4 の少ない順で分割する。**カード皆無の 2022/2026 は全チーム undefined＝従来どおり抽選**（挙動不変）。実例＝**2018 組H**: 日本(黄4) < セネガル(黄6) で日本が2位に確定（`undecided=false`）。`viewAt` がカードを含む試合を spread するためタイムライン途中スナップにもカードが渡るが、a–f が完全同値になるのは最終節後だけなので実害なし。
- 7 まで決着しない（カード無し or カードでも同値）→ **順位を捏造しない**。同順位を共有し `Standings.undecided=true` + `tiedGroupKey` を立て、UI/セルは「抽選」（🎲・斜線ハッチ）表示。`advances` はブロック全体が通過ラインに収まるときだけ true（1-2位タイ=両者通過、2-3位タイ=どちらが通過か未確定で両者 false）。
